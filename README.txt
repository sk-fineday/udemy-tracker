==============================================
 Udemy 学習トラッカー セットアップガイド
==============================================

このアプリは、Udemyコースの学習進捗を最大3コースまでタブ切り替えで管理できる
シングルHTMLアプリです。Google Sheetsと連携してPCとスマホで進捗を同期できます。

▼ 目次

  2. Google Sheets APIを有効化

----------------------------------------------
1. Google Cloud Consoleでプロジェクトを作成
----------------------------------------------
1) https://console.cloud.google.com/ にGoogleアカウントでログイン
2) 画面上部のプロジェクト選択ドロップダウン → 「新しいプロジェクト」
3) プロジェクト名（例: udemy-tracker）を入力 → 作成
4) 作成完了後、画面上部のドロップダウンで作成したプロジェクトを選択


----------------------------------------------
2. Google Sheets APIを有効化
----------------------------------------------
1) 左メニュー（ハンバーガー）→「APIとサービス」→「ライブラリ」
2) 検索ボックスに「Google Sheets API」と入力
3) 「Google Sheets API」を選択 → 「有効にする」


----------------------------------------------
3. OAuth 2.0クライアントIDの作成
----------------------------------------------
このアプリは読み書きを行うため、APIキーではなくOAuth 2.0を使用します。

【3-1. OAuth同意画面の設定】
1) 左メニュー「APIとサービス」→「OAuth同意画面」
2) ユーザータイプ → 「外部」を選択 → 作成
3) アプリ情報を入力:
     - アプリ名: 例) Udemy Tracker
     - ユーザーサポートメール: 自分のメール
     - デベロッパーの連絡先情報: 自分のメール
4) スコープは何も追加しなくてOK → 保存して次へ
5) テストユーザー → 「+ ADD USERS」で自分のGoogleメールを追加
6) 公開ステータスは「テスト」のままでOK
   （自分専用なら本番公開せず、テストモードで使うのが簡単）

【3-2. クライアントIDの作成】
1) 左メニュー「APIとサービス」→「認証情報」
2) 上部「+ 認証情報を作成」→「OAuthクライアントID」
3) アプリケーションの種類: 「ウェブアプリケーション」
4) 名前: 例) udemy-tracker-web
5) ★承認済みのJavaScriptの生成元 に以下を追加 (重要)★
     - http://localhost
     - http://localhost:8080
   ※ GitHub Pagesなど別の場所にホストする場合は、そのオリジン
     （例: https://your-name.github.io）も追加。
6) 承認済みのリダイレクトURI は不要（このアプリは Token Client 方式）
7) 「作成」をクリック
8) 表示されたクライアントID（xxxxxxxx.apps.googleusercontent.com）を
   コピーして、安全な場所にメモしておく
   ※ クライアントシークレットはこのアプリでは使いません


----------------------------------------------
4. Googleスプレッドシートの作成
----------------------------------------------
1) https://sheets.google.com/ にアクセス → 新規スプレッドシート
2) 名前を付ける（例: Udemy学習トラッカー）
3) ブラウザのURLバーを確認:
   https://docs.google.com/spreadsheets/d/【ここがスプレッドシートID】/edit#gid=0
   /d/ と /edit の間の文字列がスプレッドシートID。コピーしてメモしておく。
4) シート構造はアプリ初回起動時に自動作成されるので、手動セットアップは不要。


----------------------------------------------
5. シートの構造（自動生成される）
----------------------------------------------
■ シート: progress（コース1用）
■ シート: progress_c1（コース2用）
■ シート: progress_c2（コース3用）
  各progressシートの列構成:
  | A: dayKey | B: done | C: note | D: updatedAt |
  例) 0_0  | TRUE  | サクッと終わった  | 2026-05-10T08:30:00.000Z
       0_1  | FALSE |                |
  - dayKey: 「週index_日index」形式
  - done: TRUE / FALSE
  - note: メモ文字列
  - updatedAt: ISO日時文字列（完了日や最終更新日）

■ シート: meta
  | A: key           | B: value      |
  例) startDate      | 2026-05-10    （コース1の開始日）
      startDate_c1   | 2026-08-01    （コース2の開始日）
      startDate_c2   |               （コース3未設定）
  - 各コースの学習開始日を保存。複数端末で同期する際、シート側の値が優先される。


----------------------------------------------
6. アプリのアップロードと起動
----------------------------------------------

【方法A（推奨）: GitHub Pagesでホスト】
1) GitHubで新しいリポジトリを作成（例: udemy-tracker, Public）
2) tracker.html と tracker.css をリポジトリにコミット & プッシュ
3) リポジトリの Settings → Pages → Branch を main / (root) に設定
4) https://<your-name>.github.io/udemy-tracker/tracker.html でアクセス
5) このオリジン（https://<your-name>.github.io）を、手順3-2のステップ5
   「承認済みのJavaScriptの生成元」に追加


【方法B: ローカルでテスト】（PCで動作確認）
1) ターミナルで tracker.html があるフォルダへ移動
   例) cd ~/Desktop/Claude/udemy-tracker
2) 簡易サーバを起動:
     python3 -m http.server 8080
3) ブラウザで http://localhost:8080/tracker.html にアクセス

※ tracker.html と tracker.css は必ず同じフォルダに置いてください。


----------------------------------------------
7. 初回起動時の操作
----------------------------------------------
1) tracker.html を開くと「初回セットアップ」画面が表示される
2) 以下を入力:
   - Google OAuth クライアントID: 手順3-2で取得したID
   - スプレッドシートID: 手順4で取得したID
3) 「設定を保存して開始」をタップ
4) 「Googleでサインイン」をタップしてGoogleアカウントにログイン
   - 「このアプリは確認されていません」と警告が出る場合:
     「詳細」→「(アプリ名)に移動(安全ではないページ)」を選択
     （自分が作成したアプリなので問題ありません）
   - 権限: Google Sheetsへのアクセスを「許可」
5) アプリが progress / meta シートを自動作成し、メイン画面が表示される
6) コース1のタブが表示されるので、「開始日の設定」フォームに学習開始日を入力
   して「開始日を保存」をタップするとダッシュボードが表示される

【localStorageに保存される項目】
  - Client ID / Spreadsheet ID
  - コースごとの開始日・進捗キャッシュ・未同期キュー・表示週
HTMLファイル自体には認証情報を含めません。


----------------------------------------------
8. コースの追加方法
----------------------------------------------
コース2・コース3を追加するには、tracker.html のコードを編集します。

1) tracker.html をテキストエディタで開く
2) ファイル冒頭付近の COURSES 配列を探す:

   const COURSES = [
     { label: "Web Developer Bootcamp", weeks: [ ... ] },
     { label: "コース2", weeks: [] },   ← ここを編集
     { label: "コース3", weeks: [] },   ← ここを編集
   ];

3) 追加したいコースの label（コース名）と weeks（週ごとのタスク）を設定する
   weeks の構造は既存のコース1を参考にしてください。

4) 変更を保存してリポジトリにプッシュすると、タブにコース名が表示される
5) アプリでそのタブをタップし、「開始日の設定」フォームに開始日を入力する

【weeksの構造例】
  { label:"Week 1", theme:"テーマ説明", days:[
    {task:"タスク内容", mins:30, rest:false},
    {task:"休憩", mins:0, rest:true},
  ]},


----------------------------------------------
9. PCとスマホで同期する方法
----------------------------------------------
- 同じGoogleアカウントでログインし、同じ Client ID / Spreadsheet ID
  を使えば、PC・スマホどちらからでも同じ進捗データが見られます
- 端末ごとに最初の1回だけセットアップ画面で同じ値を入力してください
- スマホ側の OAuth 生成元: そのスマホでアプリを開くオリジンを必ず
  「承認済みのJavaScriptの生成元」に追加
  （GitHub Pagesに置くなら、そのURLのオリジンを登録すればPC/スマホ共通）


----------------------------------------------
10. データのバックアップ
----------------------------------------------
- データの正本は Googleスプレッドシート（progress / meta シート）
- バックアップは「ファイル → コピーを作成」または
  「ファイル → ダウンロード → CSV」で取得可能
- localStorageの「設定リセット」ボタンを押すとローカルキャッシュは消えますが、
  Sheets上のデータは消えません（次回サインインで復元されます）


----------------------------------------------
11. トラブルシューティング
----------------------------------------------

■ サインイン時に「invalid_client」「アクセスをブロック」
  → クライアントIDが間違っている、または「承認済みのJavaScriptの生成元」に
    現在開いているURLのオリジンが登録されていない
  → OAuth同意画面のテストユーザーに自分のメールが追加されていない

■ サインインボタンを押しても何も起きない
  → ポップアップブロッカーがブロックしていないか確認
  → ブラウザのコンソール（F12）にエラーが出ていないか確認

■ 「Sheets API 403」エラー
  → 手順2のGoogle Sheets APIが有効化されていない
  → 別のGoogleアカウントが所有するシートにアクセスしようとしている

■ 「Sheets API 404」エラー
  → スプレッドシートIDが間違っている
  → シートが削除されている

■ 開始日を変更したい
  → metaシートの該当コースの startDate セルを直接編集 → アプリを再読込
  → またはmetaシートのB列（value）を書き換えて「↻ 同期」をタップ

■ スケジュールがズレた / 完了日を直したい
  → progressシートのD列(updatedAt)を編集すると、その日付を実績として
    再スケジュールされる
  → 完了を取り消したい場合はB列をFALSEにしてアプリ「↻ 同期」をタップ

■ コース2・3のタブが「このコースはまだ設定されていません」と表示される
  → tracker.html の COURSES 配列にそのコースのweeksデータを追加してください
    （手順8参照）
